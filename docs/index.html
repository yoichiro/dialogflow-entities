Dialogflow Entities

Dialogflow にて利用可能な エンティティ のJSONファイルを掲載しています。ぜひあなたの Dialogflow エージェントにてお役立てください。

都道府県.市区町村

市区町村名を与えられると、所属する都道府県と市区町村の名前を得ることができるエンティティです。


[
  ...
  {
    "value": "埼玉県.蓮田市",
    "synonyms": ["蓮田市","蓮田"]
  },
  {
    "value": "埼玉県.坂戸市",
    "synonyms": ["坂戸市","坂戸"]
  },
  ...
]

このエンティティは、フルフィルメントにて処理されることを前提としています。例えば、Training Phraseにて "僕は ${city} で生まれました。" と登録して city をこのエンティティとした場合、ユーザが「僕は蓮田で生まれました」と発話すると、${city} として "埼玉県.蓮田市" が得られます。フルフィルメントコードにてピリオド(.)記号で文字列を分割することで、都道府県名と市区町村名を得ることが可能です。


同名の市区町村が複数の都道府県に存在する場合は、Value は複数の項目がカンマ区切りで記載されます。

{
  "value": "東京都.府中市,広島県.府中市,広島市.府中町",
  "synonyms": ["府中"]
}, {
  "value": "東京都.府中市,広島県.府中市",
  "synonyms": ["府中市"]
},{
  "value": "広島市.府中町",
  "synonyms": ["府中町"]
}

この場合、「東京都と広島県のどちらの府中市でしょうか？」といったような問いかけをユーザに行うと良いでしょう。


このエンティティは、 RESAS (地域経済分析システム) API を使って得られた情報を加工して作成されました。
